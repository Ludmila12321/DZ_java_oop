**Урок 6. ООП Дизайн и Solid**  
***Рефакторинг и\или оптимизация проекта предыдущего дз с учетом теоретических основ SOLID’а 1,2,3 - принцип.***  

Класс Phonebook теперь использует класс Contact для хранения контактов в списке contacts. Это соответствует принципу единственной ответственности, так как класс Phonebook отвечает только за управление контактами, а класс Contact отвечает только за хранение данных о контактах.

Кроме того, класс Phonebook закрыт для изменения, но открыт для расширения. Например, для добавления новой функциональности, такой как поиск контактов по номеру телефона, можно создать новый метод в классе Phonebook, не изменяя текущий код. Это соответствует принципу открытости/закрытости.

Наконец, класс Phonebook использует класс Contact в качестве родительского класса, что соответствует принципу подстановки Барбары Лисков. Дочерний класс Contact может делать все то же, что и родительский класс Object, а также имеет дополнительные методы и атрибуты для хранения данных о контакте.
